# Contributing to oscalling

Thank you for your interest in contributing to **oscalling**! We welcome contributions from the community to help make OS manipulation and system management in Python even better.

## Guidelines

1.  **Fork and Clone**: Fork the repository and clone it locally.
2.  **Create a Branch**: Create a new branch for your feature or bugfix (`git checkout -b feature/amazing-feature`).
3.  **Code Style**: Please follow PEP 8 guidelines. Clean, readable code is a priority.
4.  **Tests**: Ensure your changes are covered by tests. We aim for high reliability in system operations.
5.  **Commit Messages**: Write clear and concise commit messages.

## Pull Requests

1.  Push your branch to GitHub (`git push origin feature/amazing-feature`).
2.  Open a Pull Request against the `main` branch.
3.  Describe your changes in detail.

## Code of Conduct

Please be respectful and professional in all interactions. usage of this package implies agreement to maintain a positive community.
